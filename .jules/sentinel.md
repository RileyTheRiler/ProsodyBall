## 2025-02-28 - Preventing DOM-Based XSS via Protocol Sanitization
**Vulnerability:** Dynamically assigning untrusted URLs (e.g., `window.location.href`) directly to anchor tag `href` properties allows DOM-based Cross-Site Scripting (XSS) via dangerous protocols like `javascript:`, `data:`, or `vbscript:`.
**Learning:** `URL.createObjectURL()` outputs inherently safe `blob:` URLs, so sanitizing them is unnecessary "security theater." However, user-provided or environment-derived URLs assigned to the DOM should always be stripped of dangerous protocols using a dedicated `sanitizeUrl` helper.
**Prevention:** Implement and apply a standard `sanitizeUrl` helper (e.g., matching `/^(%20|\s)*(javascript|data|vbscript):/i`) to all dynamic `.href` assignments where the source could potentially be controlled by an attacker, while leaving system-generated secure URLs (like `blob:` URLs) untouched.
